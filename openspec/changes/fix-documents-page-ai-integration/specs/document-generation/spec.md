# Document Generation Capability

## ADDED Requirements

### Requirement: Document Generation API Endpoint
The system SHALL provide a reliable `/api/documents/process` endpoint that accepts document generation requests and returns properly formatted legal documents generated by AI.

#### Scenario: Successful document generation with basic input
- **WHEN** user submits a valid document generation request with document_type="Mietvertrag" and instructions="Erstelle einen Mietvertrag für eine 2-Zimmer-Wohnung"
- **THEN** the system returns HTTP 200 with a JSON response containing `{success: true, document: {id, title, content, ...}}`
- **AND** the document content is well-formatted HTML containing German legal text
- **AND** the response time is under 10 seconds

#### Scenario: Document generation with template
- **WHEN** user submits a document generation request with a template_id and instructions
- **THEN** the system uses the specified template as a base structure
- **AND** fills template variables with AI-generated content
- **AND** returns a complete document following template format

#### Scenario: Document generation with file upload
- **WHEN** user uploads a PDF/DOCX file and requests document generation
- **THEN** the system extracts text from uploaded file
- **AND** uses extracted content as context for AI generation
- **AND** applies PII sanitization to uploaded content
- **AND** returns document with sanitization summary in metadata

#### Scenario: Error handling for invalid requests
- **WHEN** user submits a request with missing required fields (no document_type)
- **THEN** the system returns HTTP 400 with clear error message
- **AND** error message specifies which fields are required

#### Scenario: Error handling for AI service failures
- **WHEN** AI service (Together AI) returns an error or times out
- **THEN** the system returns HTTP 503 with user-friendly error message
- **AND** error is logged to backend with full details for debugging
- **AND** user sees actionable message like "AI service temporarily unavailable. Please try again."

### Requirement: Frontend Document Generation Interface
The system SHALL provide a documents page (`/documents`) where users can input requirements and generate legal documents using AI.

#### Scenario: Generate button triggers document creation
- **WHEN** user fills in document type, instructions, and clicks "Dokument erzeugen" button
- **THEN** loading overlay displays "Dokument wird erstellt..."
- **AND** system calls `/api/documents/process` with action="generate"
- **AND** on success, displays generated document in preview pane
- **AND** shows word count and action buttons (Accept, Reject, Copy, Export)

#### Scenario: Display user-friendly errors
- **WHEN** document generation fails due to network error
- **THEN** system hides loading overlay
- **AND** displays error message "Generierung fehlgeschlagen: [reason]" in red feedback bar
- **AND** preview pane remains empty or shows previous content

#### Scenario: Handle authentication failures gracefully
- **WHEN** document generation request returns 401/403 authentication error
- **THEN** system displays specific message "Nicht autorisiert. Bitte erneut anmelden."
- **AND** user can log in again without losing entered data

### Requirement: Document Generation Response Format
The system SHALL return document generation responses in a consistent, parseable format that frontend can reliably process.

#### Scenario: Consistent response structure
- **WHEN** backend successfully generates a document
- **THEN** response contains all required fields: `{success, document: {id, title, content, document_type, metadata}, processing_state}`
- **AND** `content` field contains HTML-formatted document text
- **AND** `metadata` contains optional fields like sanitization info, model used, tokens

#### Scenario: Response includes actionable metadata
- **WHEN** document is generated with PII sanitization
- **THEN** response metadata includes `redactions: {[REDACTED_EMAIL]: 2, [REDACTED_PHONE]: 1}`
- **AND** frontend can display summary "Automatische Schwärzungen: 2× Email, 1× Telefon"

### Requirement: Document Processing Workflow
The system SHALL support a multi-step document workflow: upload → generate → review → export.

#### Scenario: Complete workflow from upload to export
- **WHEN** user uploads a file, generates document, reviews it, and exports as DOCX
- **THEN** uploaded file is stored with unique ID
- **AND** extracted text is used in AI prompt
- **AND** generated document is saved to database with user_id
- **AND** export creates downloadable DOCX file with proper formatting

#### Scenario: Template selection workflow
- **WHEN** user opens templates modal and selects "NDA – Standard (DE)"
- **THEN** document type field is pre-filled with "NDA"
- **AND** requirements field is pre-filled with template prompt
- **AND** template content is used in generation request

### Requirement: Error Recovery and Retry
The system SHALL allow users to retry failed document generation without losing input data.

#### Scenario: Retry after AI service timeout
- **WHEN** document generation times out after 60 seconds
- **THEN** error message displays "Zeitüberschreitung. Bitte erneut versuchen."
- **AND** user's document type and instructions remain filled in input fields
- **AND** user can click retry button or "Dokument erzeugen" again
- **AND** second attempt uses same inputs automatically

#### Scenario: Edit and regenerate workflow
- **WHEN** user generates a document, reviews it, wants changes
- **THEN** user can edit instructions field and click "Erneut generieren"
- **AND** system regenerates document with updated instructions
- **AND** previous document is replaced in preview pane
