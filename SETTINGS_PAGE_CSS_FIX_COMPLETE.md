These classes need to be **generated by Tailwind CSS** during the build process.

## Settings Page Structure

The Settings page uses **scoped CSS** in combination with **Tailwind utilities**:

### Custom Component Styles (in main.css)
- `.h1`, `.h2`, `.h3`, `.h4` - Typography components
- `.btn`, `.btn-primary`, `.btn-secondary` - Button styles
- `.card`, `.card-header`, `.card-body` - Card components
- `.form-input`, `.form-select`, `.form-label` - Form elements
- `.badge`, `.badge-*` - Status badges
- `.modal`, `.modal-*` - Modal dialogs
- `.tab`, `.tab-active` - Tab navigation

### Tailwind Utilities (in template)
Used for layout, spacing, and specific styling needs that complement the component styles.

## How It Works

1. **Nuxt 4** with `@nuxt/ui` provides Tailwind CSS v4 integration
2. **Build Process** scans all Vue files and generates only the classes that are actually used
3. **CSS Load Order**:
   ```
   tailwind.css (base + utilities) 
   ? main.css (custom components)
   ```
4. **Vue Scoped Styles** in `settings.vue` for page-specific layouts
